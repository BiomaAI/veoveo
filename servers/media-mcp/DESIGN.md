# Media MCP Server

The media server is the provider-backed generation surface: model discovery,
typed generation schemas, prediction execution, artifact output, and webhook
completion. Provider adapters vary internally; the user-facing MCP contract
is provider-neutral, per the repository rule that adapter details never leak
into the public surface.

## Protocol Surface

The server exposes the shared generation conventions from
`veoveo_mcp_contract`: catalog and model resources, prediction and artifact
templates, and usage records under its canonical scheme
(`ServerResourceUris`). Generation runs as task-augmented tools on the shared
task runtime; provider completion arrives through webhook waiters, and a
missing webhook delivery is an operational failure rather than a silent
retry.

## Boundaries

- Provider credentials and provider-specific request shapes stay inside the
  adapter; tools publish canonical schemas generated through the shared
  `tool` macro.
- Generated bytes land in the artifact plane with ownership and provenance;
  the server serves no bytes and keeps no private control database.
- Artifact and usage operations present the forwarded gateway internal
  identity.

## Standards And Protocols

Model Context Protocol over JSON-RPC 2.0 and Streamable HTTP; JSON Schema
2020-12 tool contracts; the final task extension per
[`mcp/task-extension`](../../mcp/task-extension); provider webhook
completion as documented in the crate.
